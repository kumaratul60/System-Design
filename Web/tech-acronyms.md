# Tech Acronyms & Specifications: A Developer's Guide

A curated list of technical abbreviations, organized by domain for easier learning and reference.

---

## 1. Core Computing & Hardware

_The fundamental building blocks of a computer._

- **CPU** — Central Processing Unit (The "brain" of the computer)
- **GPU** — Graphics Processing Unit (Specialized for graphics and parallel processing)
- **RAM** — Random Access Memory (Volatile, high-speed memory for active programs)
- **ROM** — Read-Only Memory (Non-volatile memory, for firmware like the BIOS)
- **SSD** — Solid State Drive (Fast, non-volatile storage)
- **HDD** — Hard Disk Drive (Slower, mechanical non-volatile storage)
- **ALU** — Arithmetic Logic Unit (Performs arithmetic and logic operations in the CPU)
- **BIOS** — Basic Input/Output System (Firmware to initialize hardware during boot)
- **UEFI** — Unified Extensible Firmware Interface (Modern replacement for BIOS)

---

## 2. OS, Interfaces & Development Tools

_How we interact with computers and write software._

- **OS** — Operating System
- **CLI** — Command Line Interface
- **GUI** — Graphical User Interface
- **SDK** — Software Development Kit
- **IDE** — Integrated Development Environment
- **VM** — Virtual Machine (A software-based emulation of a computer)

---

## 3. Networking & Protocols

_How computers and services communicate over a network._

- **IP** — Internet Protocol (Core protocol for addressing and routing packets)
- **TCP** — Transmission Control Protocol (Connection-oriented, reliable data transfer)
- **UDP** — User Datagram Protocol (Connectionless, faster, but less reliable)
- **HTTP** — HyperText Transfer Protocol (The foundation of data communication for the Web)
- **HTTPS** — HyperText Transfer Protocol Secure (HTTP with a security layer, usually TLS)
- **URL** — Uniform Resource Locator (A specific web address)
- **URI** — Uniform Resource Identifier (A broader standard for identifying resources)
- **DNS** — Domain Name System (Translates domain names to IP addresses)
- **FTP** — File Transfer Protocol
- **SMTP** — Simple Mail Transfer Protocol (For sending email)
- **SSH** — Secure Shell (For secure remote login and other network services)
- **CDN** — Content Delivery Network (Distributes content globally to improve performance)

## 6. Data & Databases

_How we store, query, and manage data._

- **DB** — Database
- **DBMS** — Database Management System
- **RDBMS** — Relational Database Management System
- **SQL** — Structured Query Language
- **NoSQL** — "Not Only SQL" (Databases that are not primarily relational)
- **ORM** — Object-Relational Mapping (A technique to convert data between incompatible type systems)
- **ETL** — Extract, Transform, Load (A data integration process)
- **ACID** — Atomicity, Consistency, Isolation, Durability (Properties of reliable database transactions)
- **OLTP** — Online Transaction Processing (Systems for managing transactional applications)
- **OLAP** — Online Analytical Processing (Systems for analyzing large amounts of data)
- **CRUD** — Create, Read, Update, Delete (The four basic database operations)

## 7. APIs & Data Exchange

_How different applications and services communicate with each other._

- **API** — Application Programming Interface
- **REST** — Representational State Transfer (An architectural style for APIs)
- **SOAP** — Simple Object Access Protocol (A protocol for exchanging structured information)
- **GraphQL** — Graph Query Language (A query language and runtime for APIs)
- **gRPC** — Google Remote Procedure Call (A high-performance RPC framework)
- **JSON** — JavaScript Object Notation
- **XML** — eXtensible Markup Language
- **YAML** — YAML Ain’t Markup Language
- **TOML** — Tom’s Obvious Minimal Language
- **Proto** — Protocol Buffers (Google's data serialization format)
- **OpenAPI (OAS)** — OpenAPI Specification (A standard for describing RESTful APIs)
- **WSDL** — Web Services Description Language (For describing SOAP-based web services)
