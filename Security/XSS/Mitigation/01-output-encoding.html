Output Encoding (Text vs. HTML)

This is the #1 defense against XSS. Treating data as text prevents the browser from executing it.

Vulnerable (innerHTML):
Secure (textContent):

Why this works:

textContent tells the browser: "This is just a string of words, don't look for tags." Even if the user sends <script>, the browser will just draw the letters on the screen instead of running the code.

Try this: Append ?name=<img src=x onerror=alert(1)> to the URL. The red box will trigger an alert, but the green box will safely show the text.
